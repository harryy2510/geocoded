This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Geo API ‚Äî a Cloudflare Worker serving country, state, city, and location data from KV storage. Built with Hono and deployed via Wrangler.

## Commands

- `bun dev` ‚Äî start local dev server (wrangler dev)
- `bun run deploy` ‚Äî deploy to Cloudflare
- `bun seed` ‚Äî generate KV bulk JSON files from upstream data into `data/`
- `bun seed:upload` ‚Äî generate + upload to Cloudflare KV (reads namespace ID from `wrangler.jsonc`)
- `bun run types` ‚Äî regenerate `worker-configuration.d.ts` (auto-generated, do not edit)
- `bunx tsc --noEmit` ‚Äî type-check

## Architecture

**Runtime**: Cloudflare Worker with Hono router (`src/index.ts`)
**Storage**: Cloudflare KV (`GEO_KV` binding) with these key patterns:

- `countries` ‚Äî all countries array
- `states:{COUNTRY_ISO2}` ‚Äî states for a country
- `cities:{COUNTRY_ISO2}:{STATE_ISO2}` ‚Äî cities for a country+state
- `apikey:{KEY}` ‚Äî valid API keys (value: `{ name, email, createdAt }`)
- `email:{EMAIL}` ‚Äî maps email to API key (for dedup)

**Secrets**: `RESEND_API_KEY` ‚Äî Resend API key for sending registration emails (set via `wrangler secret put RESEND_API_KEY`)

**Data pipeline**: `scripts/seed.ts` fetches from [dr5hn/countries-states-cities-database](https://github.com/dr5hn/countries-states-cities-database), maps all fields to camelCase (`src/types.ts`), and writes bulk JSON files. Pass `--upload` to also push to KV (uses `--binding GEO_KV` so the namespace ID is read from `wrangler.jsonc`). A GitHub Actions workflow (`.github/workflows/seed.yml`) runs `bun seed:upload` on pushes to `scripts/seed.ts`.

**Authentication**: All API routes (except `GET /`, `GET /register`, `GET /openapi.json`) require an `Authorization: Bearer <key>` header. Keys are validated against `apikey:{key}` in KV. Requests with a `Referer` starting with `https://geocoded.me` are exempt (docs page "Try It" links).

**API routes** (all return JSON with aggressive cache headers):

- `GET /` ‚Äî Scalar interactive API docs (no auth required)
- `GET /openapi.json` ‚Äî OpenAPI 3.1 spec (no auth required)
- `GET /register` ‚Äî registration form page (no auth required)
- `POST /register` ‚Äî API key registration endpoint (no auth required)
- `GET /location` ‚Äî caller's geo info from Cloudflare `cf` properties, enriched with full country/state/city details from KV (`countryInfo`, `stateInfo`, `cityInfo`)
- `GET /countries` / `GET /countries/:id` ‚Äî lookup by iso2, iso3, or name
- `GET /countries/:country/states` / `‚Ä¶/states/:state`
- `GET /countries/:country/states/:state/cities` / `‚Ä¶/cities/:city`

All endpoints support an optional `?fields=` query parameter (comma-separated) to return only specific fields, e.g. `?fields=name,iso2`. Dot notation is supported for nested objects, e.g. `?fields=ip,countryInfo.name,countryInfo.iso2`. When omitted, all fields are returned.

## Code Conventions

- **Package manager**: bun (never npm/yarn)
- **Formatting**: Prettier ‚Äî tabs, single quotes, no semicolons, trailing commas (see `.prettierrc`)
- **Types**: `type` keyword only, no `interface`. Inline type imports: `import { type Foo } from './bar'`
- **Auto-generated files**: never edit `worker-configuration.d.ts`
- **Data files**: `data/*.json` is gitignored (generated by seed script)
